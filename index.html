How It Works

ChatSphere is a real-time communication platform with admin-controlled room access.

Admin-Controlled Access

Room Codes: Admins create secure room codes for private chat rooms.
Approval System: Users request to join, admins approve/reject in real-time.
User Management: Admins can remove users at any time.

Real-Time Communication

Uses WebSocket protocol via Socket.IO for bi-directional messaging, ensuring instant message delivery without polling or page refreshes.

Technology Stack

Backend: Node.js + Express.js
Real-Time: Socket.IO
Frontend: Vanilla JavaScript (ES6+)

Features

• Automatic port fallback (handles port conflicts)
• Graceful disconnect handling
• Real-time notifications
• Dynamic user list updates

Documentation

Getting Started

ChatSphere uses a simple room-based architecture with admin controls. To get started:

For Admins:

  • Click "Create Room" and generate a unique room code
  • Share the room code with participants
  • Approve or reject join requests as they come in
  • Manage users and moderate the conversation

For Users:

  • Enter the room code provided by the admin
  • Wait for admin approval
  • Start chatting once approved

System Architecture

Server Side: Built with Node.js and Express.js, the server handles HTTP requests and WebSocket connections through Socket.IO.

Real-Time Layer: Socket.IO manages bidirectional communication between clients and server using WebSocket protocol with automatic fallback mechanisms.

Client Side: Vanilla JavaScript (ES6+) provides a lightweight, fast interface without framework overhead.

Key Events

The application uses Socket.IO events for real-time communication:

  • create-room - Admin creates a new room
  • join-request - User requests to join a room
  • approve-user - Admin approves a join request
  • reject-user - Admin rejects a join request
  • send-message - Send a message to the room
  • user-disconnected - Handle user disconnections
  • remove-user - Admin removes a user from room

Technical Features

  • Port Fallback: Automatically tries alternative ports if the default is occupied
  • Graceful Disconnection: Handles network interruptions and reconnection attempts
  • Real-Time Notifications: Instant updates for join requests, approvals, and user actions
  • Dynamic Updates: User lists and room states update automatically without page refresh
  • Room Isolation: Each room is completely isolated with separate user lists and message streams
